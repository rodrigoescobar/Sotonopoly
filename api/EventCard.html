EventCard

Back to menu

Models the "Chance" and "Community Chest" cards. Each instance of this class represents a card from either deck. This class has no methods available; all data processing occurs in the GameUpdater

Datastore model entity. Inherits from Game entity.

Variables

text (String)

The text shown on the card (e.g. "Proceed to Zepler").

Default value: "card"

Required? Yes.

event (integer)

An integer encoding what the value means:

  • 0: Income - value is an int, player recieves (or loses, if value is negative) that much money
  • 1: Move - value is an int, player moves forwards (or backwards, if value is negative) that many spaces
  • 2: MoveTo - value is an int in range 0-39 (inclusive), player moves to that location on the board
  • 3: PayAll - value is an int, player pays (or receives, if value is negative) that much money to/from each player
  • 4: Jail - value is int, 1 = GoToJail, 0 = GetOutOfJail
  • 5: Utilty - value is not important (= 0), just move to the nearest Utility
  • 6: Station - value is not important (= 0), just move to the nearest Station
  • 7: Pay Per Property - value is int to pay per house/office owned, value2 is int to pay per hotel/lab owneed

Default value: 0

Required? Yes.

value (integer)

An integer value storing a quantity relevant depending on event.

Default value: 0

Required? Yes.

value2 (integer)

An integer value storing the cost for labs for Pay Per Property cards. This value is not required and redundant for all other event cards.

Default value: 0

Required? No.

drawn (boolean)

True - this card has been drawn.

False - this card has not yet been drawn

Default value: False

Required? Yes.

isChance (boolean)

True - this is a Chance card.

False - this is a Community Chest card.

Default value: False.

Required? Yes.
