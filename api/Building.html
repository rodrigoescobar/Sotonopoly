Building

Back to menu

This class stores data associated with a building on the board. This class has no methods available; all data processing occurs in the GameUpdater

Datastore model entity. Inherits from Game entity.

Variables

name (String)

The name of the property (e.g. "Zepler")

Required? Yes

position (integer)

An integer with range 0-39 inclusive. Signifies the place on the board this building is located (e.g. Avenue Campus will have position 1).

Required? Yes

type (String enum: "building", "shop", "pub")

A field to store what type of building. In contrast to classic Monopoly: building is property, shop is utility, pub is station.

Default value: "building".

Required? Yes

colour (String)

The hex value of the colour of the property (used to construct the "card" displayed to the user). A String value to meet the requirements of the Google App Engine Datastore.

Required? Yes

cost (integer)

The cost to buy the property.

Required? Yes

houses (integer)

The number of houses (offices) currently on the property of range 0-5 (5 is equivilant to a research lab [hotel]).

Default value: 0

Required? Yes

owner (Google UserProperty)

Handle to the user who owns the property.

Required? No

rent (list<integer>)

Each element in the list stores the cost of rent for x number of offices:

  • rent[0] - rent with no offices;
  • rent[1] - rent with 1 office;
  • rent[2] - rent with 2 offices;
  • rent[3] - rent with 3 offices;
  • rent[4] - rent with 4 offices;
  • rent[5] - rent with a research lab.

buildCost (integer)

The cost to add offices or research labs (houses or hotels)

Required? No

mortgaged (boolean)

True - this property is mortgaged

False - this property is not mortgaged

Default value: False.

Required? Yes
